URDF & Xacro Modeling for Humanoid Robots

This chapter delves into the Unified Robot Description Format (URDF) and its powerful extension, Xacro. You will learn how to create detailed and flexible models of humanoid robots, specifying their kinematics, dynamics, visual properties, and collision geometries.

URDF Fundamentals​

  • Links: Defining rigid bodies of the robot.
  • Joints: Specifying the connections and motion constraints between links.
  • Visuals: Adding visual meshes for rendering.
  • Collisions: Defining collision geometries for physics simulation.
  • Inertial Properties: Mass, inertia matrix for accurate dynamics.

Xacro for Enhanced Modeling​

  • Macros: Reusing common URDF structures.
  • Properties: Defining and using variables within your models.
  • Includes: Modularizing your robot descriptions.
  • Mathematical Expressions: Using basic math for calculations (e.g., joint limits).

Modeling a 20+ DoF Humanoid​

  • Kinematic Chains: Structuring the robot's limbs and torso.
  • Degrees of Freedom (DoF): Understanding and modeling complex humanoid movements.
  • Materials and Textures: Enhancing visual realism.

Example: Humanoid Leg Segment (using Xacro Macros)​

Instead of repeating code for each limb, Xacro allows us to define reusable macros. Here's how you can define and use a macro for an arm segment, allowing you to easily add multiple arms with similar structures.

humanoid_macro_example.xacro​

<?xml version="1.0"?>
<robot name="humanoid_with_macros" xmlns:xacro="http://www.ros.org/wiki/xacro">

  <link name="base_link">
    <visual>
      <geometry>
        <box size="0.1 0.1 0.2"/>
      </geometry>
    </visual>
    <collision>
      <geometry>
        <box size="0.1 0.1 0.2"/>
      </geometry>
    </collision>
    <inertial>
      <mass value="1.0"/>
      <inertia ixx="0.01" ixy="0.0" ixz="0.0" iyy="0.01" iyz="0.0" izz="0.01"/>
    </inertial>
  </link>

  <xacro:macro name="arm_segment" params="prefix parent_link_name origin_xyz">
    <link name="${prefix}_arm_link">
      <visual>
        <geometry>
          <box size="0.05 0.05 0.15"/>
        </geometry>
      </visual>
      <collision>
        <geometry>
          <box size="0.05 0.05 0.15"/>
        </geometry>
      </collision>
      <inertial>
        <mass value="0.5"/>
        <inertia ixx="0.001" ixy="0.0" ixz="0.0" iyy="0.001" iyz="0.0" izz="0.001"/>
      </inertial>
    </link>

    <joint name="${prefix}_joint" type="revolute">
      <parent link="${parent_link_name}"/>
      <child link="${prefix}_arm_link"/>
      <origin xyz="${origin_xyz}" rpy="0 0 0"/>
      <axis xyz="0 1 0"/>
      <limit lower="-1.57" upper="1.57" effort="100" velocity="10"/>
    </joint>
  </xacro:macro>

  <xacro:arm_segment prefix="right" parent_link_name="base_link" origin_xyz="0.0 -0.075 0.15"/>
  <xacro:arm_segment prefix="left" parent_link_name="base_link" origin_xyz="0.0 0.075 0.15"/>

</robot>

How to Use​

  1. Save the above content as humanoid_macro_example.xacro in your ROS 2 package (e.g., examples/ros2_humanoid_control_example/humanoid_control/urdf/).
  2. To process this Xacro file into a standard URDF, use the following command: 
    ros2 run xacro xacro --inorder humanoid_macro_example.xacro > robot.urdf
  3. You can then visualize the generated robot.urdf in RViz2.
