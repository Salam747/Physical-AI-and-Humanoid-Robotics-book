# Creating and Publishing Custom ROS 2 Messages

This chapter guides you through the process of defining, building, and using custom message types in ROS 2. Custom messages are essential for exchanging domain-specific data between nodes in your robotics application.

## Defining Custom Messages

*   **Message Definition Files (`.msg`)**: Syntax and best practices for creating `.msg` files.
*   **Data Types**: Understanding primitive types, arrays, and nested messages.
*   **Header**: Including `std_msgs/msg/Header` for timestamped data.

## Building Custom Message Packages

*   **Package Structure**: Organizing your custom message definitions within a ROS 2 package.
*   **`CMakeLists.txt` and `package.xml`**: Configuring the build system for custom messages.
*   **`colcon build`**: Compiling your custom message package.

## Publishing and Subscribing to Custom Messages

*   **Python (`rclpy`) Examples**: Demonstrating how to publish and subscribe to your custom messages using `rclpy`.
*   **C++ (`rclcpp`) Examples**: (Optional) Demonstrating custom message usage in C++.

## Example: Custom Robot Status Message

We will create a custom message type called `RobotStatus` to convey information about our robot's operational state.

### 1. Define the Custom Message (`RobotStatus.msg`)

First, define the structure of our `RobotStatus` message in a `.msg` file.

```
# examples/ros2_custom_message_example/robot_status_msgs/msg/RobotStatus.msg
# Contains information about the robot's current status

std_msgs/msg/Header header
uint8 status_code
float32 battery_level
string message
```

Here:
*   `std_msgs/msg/Header`: Includes a timestamp for when the status was recorded.
*   `uint8 status_code`: A numerical code indicating the robot's state (e.g., 0 for normal, 1 for warning, 2 for error).
*   `float32 battery_level`: The current battery charge as a percentage.
*   `string message`: A human-readable message describing the status.

### 2. Configure ROS 2 Package for Custom Message

To make ROS 2 recognize and build our custom message, we need a standard ROS 2 package.

Create the package structure:
```bash
mkdir -p examples/ros2_custom_message_example/robot_status_msgs/msg
```
(This has already been performed)

#### `package.xml`

This file declares our package and its dependencies, including `rosidl_default_generators` which is needed to build messages.

```xml
<!-- examples/ros2_custom_message_example/robot_status_msgs/package.xml -->
<?xml version="1.0"?>
<?xml-model href="http://download.ros.org/schema/package_format3.xsd" schematypens="http://www.w3.org/2001/XMLSchema"?>
<package format="3">
  <name>robot_status_msgs</name>
  <version>0.0.0</version>
  <description>Custom messages for robot status</description>
  <maintainer email="user@example.com">user</maintainer>
  <license>Apache-2.0</license>

  <buildtool_depend>ament_cmake</buildtool_depend>
  <buildtool_depend>rosidl_default_generators</buildtool_depend>

  <depend>rclpy</depend>
  <depend>std_msgs</depend>
  <member_of_group>rosidl_interface_packages</member_of_group>

  <export>
    <build_type>ament_cmake</build_type>
  </export>
</package>
```

#### `CMakeLists.txt`

This file instructs `colcon` how to build the message interface.

```cmake
# examples/ros2_custom_message_example/robot_status_msgs/CMakeLists.txt
cmake_minimum_required(VERSION 3.8)
project(robot_status_msgs)

if(CMAKE_COMPILER_ID STREQUAL "GNU")
  add_compile_options(-Wall -Wextra -Wpedantic)
endif()

find_package(ament_cmake REQUIRED)
find_package(rosidl_default_generators REQUIRED)
find_package(std_msgs REQUIRED)

rosidl_generate_interfaces(${PROJECT_NAME}
  "msg/RobotStatus.msg"
  DEPENDENCIES std_msgs
)

ament_export_dependencies(rosidl_default_runtime std_msgs)

ament_package()
```

### 3. Build the Custom Message Package

Navigate to the root of your workspace (assuming `examples/ros2_custom_message_example/` is within your ROS 2 workspace `src` directory or directly in the workspace root) and build:

```bash
# From your ROS 2 workspace root, assuming examples/ros2_custom_message_example/ is in src/
colcon build --packages-select robot_status_msgs
```

Then, source your workspace to make the message available:
```bash
source install/setup.bash
```

### 4. Create Python Publisher and Subscriber

Now that the message is built, we can use it in Python nodes.

#### Publisher Node (`custom_status_publisher.py`)

```python
# examples/ros2_custom_message_example/custom_status_publisher.py
import rclpy
from rclpy.node import Node
from rclpy.time import Time
from robot_status_msgs.msg import RobotStatus
from std_msgs.msg import Header

class CustomStatusPublisher(Node):

    def __init__(self):
        super().__init__('custom_status_publisher')
        self.publisher_ = self.create_publisher(RobotStatus, 'robot_status', 10)
        timer_period = 1.0  # seconds
        self.timer = self.create_timer(timer_period, self.timer_callback)
        self.status_code = 0

    def timer_callback(self):
        msg = RobotStatus()
        msg.header = Header()
        msg.header.stamp = self.get_clock().now().to_msg()
        msg.status_code = self.status_code % 3  # Cycle through 0, 1, 2
        msg.battery_level = 100.0 - (self.status_code * 5.0) # Example battery level decrease
        if msg.status_code == 0:
            msg.message = "Robot operating normally."
        elif msg.status_code == 1:
            msg.message = "Low battery warning."
        else:
            msg.message = "System error detected."
        
        self.publisher_.publish(msg)
        self.get_logger().info(f'Publishing: "{msg.message}" (Battery: {msg.battery_level:.1f}%)')
        self.status_code += 1

def main(args=None):
    rclpy.init(args=args)
    custom_status_publisher = CustomStatusPublisher()
    rclpy.spin(custom_status_publisher)
    custom_status_publisher.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

#### Subscriber Node (`custom_status_subscriber.py`)

```python
# examples/ros2_custom_message_example/custom_status_subscriber.py
import rclpy
from rclpy.node import Node
from robot_status_msgs.msg import RobotStatus

class CustomStatusSubscriber(Node):

    def __init__(self):
        super().__init__('custom_status_subscriber')
        self.subscription = self.create_subscription(
            RobotStatus,
            'robot_status',
            self.listener_callback,
            10)
        self.subscription  # prevent unused variable warning

    def listener_callback(self, msg):
        self.get_logger().info(f'Received RobotStatus: Timestamp={msg.header.stamp.sec}.{msg.header.stamp.nanosec}, '
                               f'Status Code={msg.status_code}, Battery={msg.battery_level:.1f}%, Message="{msg.message}"')

def main(args=None):
    rclpy.init(args=args)
    custom_status_subscriber = CustomStatusSubscriber()
    rclpy.spin(custom_status_subscriber)
    custom_status_subscriber.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

### 5. How to Run

1.  Place the `robot_status_msgs` package and the two Python scripts (`custom_status_publisher.py`, `custom_status_subscriber.py`) into the `src/` directory of a new or existing ROS 2 workspace.
2.  From the root of your ROS 2 workspace, build the custom message package:
    ```bash
    colcon build --packages-select robot_status_msgs
    ```
3.  Source your workspace:
    ```bash
    source install/setup.bash
    ```
4.  Open two separate terminal windows, source your workspace in each.
5.  Run the publisher:
    ```bash
    python3 examples/ros2_custom_message_example/custom_status_publisher.py
    ```
6.  In the other terminal, run the subscriber:
    ```bash
    python3 examples/ros2_custom_message_example/custom_status_subscriber.py
    ```
You should see the subscriber receiving the custom `RobotStatus` messages.
